## flue-agent (`apps/flue-agent/`)
The main application. Uses the [Flue](https://flue.dev) framework to define **agents** (Durable Objects) and **workflows** on Cloudflare Workers.

### Source layout
```
src/
  agents/           ← defineAgent() exports → Durable Objects
  workflows/        ← defineWorkflow() exports → durable task chains
  index.ts          ← entrypoint (currently a stub)
```

### Agent patterns
- Each agent file exports a `default defineAgent()` and optionally a named `route` (route handler).
- Each workflow file exports a `default defineWorkflow()` and optionally a named `route`.
- Models are configured per-agent (e.g. `deepseek/deepseek-v4-flash`).
- Workflow inputs use Valibot schemas.

### Flue build pipeline (critical — do NOT bypass)
The Flue CLI (`@flue/cli`) wraps Vite for bundling and auto-generates worker config:

- `bun run dev` → `flue dev --target cloudflare` (local wrangler dev)
- `bun run build` → `flue build --target cloudflare` (outputs to `dist/flue_agent/`)
- `bun run deploy` → `flue build --target cloudflare && wrangler deploy` (runs build automatically)

**Do not** run `wrangler dev` or `wrangler deploy` directly on the source — use the Flue CLI commands.

The file `.flue-vite.wrangler.jsonc` is auto-generated by Flue during dev/build. Do not edit it manually.

### wrangler.jsonc
Defines Durable Object exports and bindings. The DO class names (`FlueHelloWorldAgent`, `FlueRegistry`, `FlueTravelAgent`, `FlueTranslateWorkflow`) must match what Flue generates. If you add or remove agents/workflows, re-run `flue build` to regenerate the config.

### Environment
The app requires a `DEEPSEEK_API_KEY` in `.env`. Do not commit `.env` (it is gitignored at root level).
